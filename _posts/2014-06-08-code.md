<p>The general solution of injecting javascript into a page from a bookmark, is very usefull for many applications, especially if when combined with javascript online hosting, i explain most of the solution i used in <a href="/al3xandr3/al3xandr3.github.com/blob/master/javascript-bookmarks.html">javascript bookmarks</a>.</p>

<h2>
<a name="user-content-tablequery" class="anchor" href="#tablequery"><span class="octicon octicon-link"></span></a>Table.query</h2>

<p>A small library that given a set of data it transparently inserts it into a sqlite database and then allows for very flexible querying (SQL). Usefull for ad-hoc data tasks that read data from some file or api that require quick analysis. Adding data into a sqlite database will also allow for the data to be bigger than memory can fit. Inspired by R's <a href="http://code.google.com/p/sqldf/"><strong>sqldf</strong></a>.</p>

<p><a href="http://al3xandr3.github.io/table-query.html">code</a> - <a href="http://al3xandr3.github.io/table-query.html">web</a> - October 2013 - Tools: Ruby</p>

<h2>
<a name="user-content-cov3---custom-web-crawler" class="anchor" href="#cov3---custom-web-crawler"><span class="octicon octicon-link"></span></a>cov3 - Custom Web Crawler</h2>

<p>A clojure, selenium based web crawler with a couple of variations that i found
useful. Cov3 crawls pages with the intention of executing
(javascript)tests / validations on each of the pages it opens. The original idea
of a web crawler is that given an initial page it keeps on following all links
it finds in the same domain until all pages are visited. I added a couple of
variations: Given a sitemap.xml it randomly samples part of it(useful for big
sitemaps) and follows those urls executing some test. And another option is a
csv file, with the urls and tests to execute on each.</p>

<p><a href="https://github.com/al3xandr3/cov3">code</a> - <a href="/al3xandr3/al3xandr3.github.com/blob/master/clojure-selenium2-crawler-cov3.html">web</a> - May 2010 - Tools: clojure, selenium</p>

<p><br></p>

<h2>
<a name="user-content-jquery-twitter-mini-plugin" class="anchor" href="#jquery-twitter-mini-plugin"><span class="octicon octicon-link"></span></a>jQuery Twitter 'mini' plugin</h2>

<p>A jQuery plugin to display a twitter feed on a web page(blog for example). The
twitter api feed, i found, has an annoying particularity of not always
returning the requested number of tweets, so i added a feature that until it
gets the right amount of messages it keeps on fetching. Thats useful when you
want to fill a web page space with exactly 3 messages. Also included a "time
ago" date calculations based of john resig <a href="http://ejohn.org/blog/javascript-pretty-date/">code</a>, with an added local
timezone shift.</p>

<p><a href="https://github.com/al3xandr3/jquery-twitter-plugin">code</a> - <a href="/al3xandr3/al3xandr3.github.com/blob/master/jquery-twitter-plugin.html">web</a> - April 2010 - Tools: javascript, jquery</p>

<p><br></p>

<h2>
<a name="user-content-the-daily-extract" class="anchor" href="#the-daily-extract"><span class="octicon octicon-link"></span></a>The Daily Extract</h2>

<p>Scripts that automate the process of aggregating data in a database,
extracting it, outputting results into csv files and sending them over by
sftp. Very feature rich: task list queue, full logging, email reports, sql
templating, sftp support, extract volume validation, unit testing. And with a
strong convention over configuration theme.</p>

<p>In Aug 2009 i did a somewhat similar 2nd version of the daily extract, but
this time using clojure, with added features: web front end for task queue
manipulation, statistics on extracted volumes and sqlite for storing the task
list queue and statistics over time.</p>

<p><a href="https://github.com/al3xandr3/daily-extract">code</a> - 2007&amp;2009 - Tools: ruby, jruby, clojure</p></article>
